\section{Introduction}

\textit{Web browsers} are central to modern computing,
  enabling applications that correspond to
  over \textbf{80\%} of daily work in some organizations%
  ~\cite{montenegro2025workforce}.
Browser makers therefore invest heavily
  in browser correctness,%
  ~\cite{google-project-zero,mozilla-mythos,w3c,whatwg,wpt,interop},
  including through fuzzing~\cite{clusterfuzz},
  which uses a large number of randomly generated inputs
  to identify bugs in browsers.

Browser developers use fuzzers to find both
  security and correctness bugs.
On the security side, fuzzers such as
  Domato, Grammarinator, Fuzzowski, and MFFA%
  ~\cite{domato,grammarinator,fuzzowski,mffa}
  detect crashes in parsers, network libraries, and media engines,
  which are performance-intensive
  but also exposed to untrusted input.
On the correctness side, fuzzers such as
  R2Z2~\cite{song2022r2z2}, Janus~\cite{zhou2025janus}
  and Metamong~\cite{song2023metamong}
  use screenshots to detect browser rendering inconsistencies
  between different versions of the same browser,
  or between renderings of the different pages in the same browser,
  that may indicate a regression or rendering bug.
However, all of these approaches struggle to
  precisely identify \emph{under-invalidation bugs},
  a subtle yet critical type of correctness bug
  in browser layout engines.

\emph{Invalidation} refers to the process by which
  the browser updates internal data structures
  in response to style changes~\cite{panchekha2024reusing},
  including those triggered by JavaScript,
  user action, or animations.
Since style changes are frequent
  (60 or even 120 times per second during animations),
  substantial research has been devoted
  to designing efficient invalidation procedures%
  ~\cite{megatron,liu2023medea}.
Generally, browsers attempt to reuse \emph{as much}
  of their internal data structures as possible, to save time.
However, reusing \emph{too much} of this internal state
  leads to an incorrect rendering that mixes
  fresh (updated) and stale (reused) data;
  this is called an \emph{under-invalidation bug}.
Updating exactly the right set of fields requires
  reasoning about transitive dependencies between layout algorithms,
  reasoning that is difficult enough that public bug trackers
  for Chrome, Firefox, and Safari's layout engines
  contain numerous under-invalidation bugs.
Modern invalidation algorithms like LayoutNG~\cite{kilpatrick2021layoutng}
  are purposefully designed to reduce the risk of under-invalidation,
  and promising algorithms~\cite{mozilla2006reflow} have failed
  precisely because under-invalidation bugs proved
  difficult to find and fix.
In other words, finding under-invalidation bugs is
  both difficult and important.

To combat these issues, this paper introduces
  a new oracle that detects under-invalidation bugs
  via self-differential testing,
  and validate this oracle on bugs mined from
  real-world browser bug databases.
We extend this basic idea into \toolname, a fuzzer
  that combines this oracle with random generation
  and minimization of complex web pages and web page changes,
  and leverages automated bug clustering
  to reduce the bug review burden.
We demonstrate that \toolname is effective at
  both rapidly finding under-invalidation bugs
  (\NumBugsChromeWeights in Chrome and \NumBugsFfxWeights in Firefox)
  and minimizing and clustering them,
  reducing the style-element review volume by approximately
  \StyleElementOriginalToClusteredFactor{}.

\medskip
\noindent
This paper makes the following contributions:
\begin{itemize}
    \item A new class of bugs, \emph{under-invalidation bugs},
      that can be efficiently detected using self-differential testing;
    \item \toolname, a targeted fuzzer
      that generates web pages and web page changes,
      detects under-invalidation bugs with self-different testing,
      and minimizes and clusters the resulting bugs to reduce review load.
    \item An evaluation of \toolname on both Chrome and Firefox,
      measuring bug discovery, minimization, and clustering
      with both direct and ablation studies.
\end{itemize}
